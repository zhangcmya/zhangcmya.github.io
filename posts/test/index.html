使用hugo+LoveIt主题搭建个人博客

简介

博客网站是每个程序员必不可少的。因为博客如同我们的简历，可以充分让面试官或同事对你进一步的了解。同时书写博客可以让你总结自己学到的知识，加深对知识的掌握。那么接下来就开始了解如何使用Hugo + LoveIt主题 + valine评论进行我们博客的搭建。本文以Mac系统为例，其它的系统参考官网文档。

一. Hugo

Step1. 安装

  1. 使用Homebrew安装

    brew install hugo

  2. 使用命令查看是否安装完成

    hugo version

    出现hugo v0.89.4+extended darwin/amd64 BuildDate=unknown表示安装完成

Step2. 创建项目

使用new命令创建myBlog，命令如下：

hugo new site myBlog

![image-20211231101622063](/Users/apple/Library/Application Support/typora-user-images/image-20211231101622063.png)

创建完成后，根目录myBlog包含以下文件

.
├── archetypes: default.md是生成博文的模版
├── assets # 存放被 Hugo Pipes 处理的文件
├── content # 存放markdown文件作为博文内容
├── data # 存放 Hugo 处理的数据
├── layouts # 存放布局文件
├── static # 存放静态文件 图片 CSS JS文件
├── themes: 存放不同的主题
└── config.toml: 博客配置文件支持 JSON YAML TOML 三种格式配置文件

Step3. 新建博客

使用new命令创建test.md博客，命令如下：

hugo new posts/test.md

输入命令后，会在content文件夹下生成一个test.md文件，文件内容如下：

---
title: "Test"
date: 2018-08-05T20:56:47+08:00
draft: true
---

两条---间的内容为博客文章的配置信息，默认会生成title，date和draft，说明如下：

自动生成的内容：

  • title: # 博客标题
  • date: # 博客创建时间
  • draft: # 是否为草稿，如果为true需要在命令行加入-D 或者 --buildDrafts参数才会生成这个文档

需要手动添加的内容：

  • description: # 描述
  • tags: # 标签，用于文章分类
  • categories # 指定文章的类别

自动生成和执行添加的内容并不是绝对的，你可以根据自己的喜好配置模板文件

archetypes/default.md

总结：这个命令会在 content 目录下建立 post 目录，并在 post 下生成 test.md 文件，博文书写就在这个文件里使用 Markdown 语法完成。博文的 front matter 里draft 选项默认为 true，需要改为 false 才能发表博文，建议直接更改上面说的archetypes 目录下的 default 文件，把 draft: true 改为 draft: false，这样生成的博文就是默认可以发表的。

在博文的 front matter 里面，使用 tags 指定文章的 tag，使用categories 指定文章的类别，示例如下：

tags: [LaTeX, Hexo, font]
categories: [Mac, Linux]

Step4. 生成网页

为了本地预览博客效果，运行以下命令，即可在http://localhost:1313/中查看博客的网页效果。

hugo server
hugo server --buildDrafts -w

参数说明：

  • --buildDrafts: 生成被标记为 「草稿」 的文档
  • -w: 监控更改，如果发生更改直接显示到博客上 (非常有用，这也是我最喜欢的一点特性了)

但此时只能在本地访问 (相当于预览博客，如果与期望值不符，可以随时更改)，如果想发布到 Github Pages 上需要先将文章配置信息中的 draft: 改为 false 然后执行命令

hugo

此时你的博客目录下会多出一个 public 文件夹来。这便是 Hugo 生成的网站

二. 使用LoveIt主题

Step1. 安装主题

LoveIt 主题的仓库是: https://github.com/dillonzq/LoveIt.

你可以下载主题的 最新版本 .zip 文件 并且解压放到 themes 目录.

另外, 也可以直接把这个主题克隆到 themes 目录:

git clone https://github.com/dillonzq/LoveIt.git themes/LoveIt

或者, 初始化你的项目目录为 git 仓库, 并且把主题仓库作为你的网站目录的子模块:

git init
git submodule add https://github.com/dillonzq/LoveIt.git themes/LoveIt

Step2. 基础配置

修改config.toml

以下是LoveIt主题的基本配置：

baseURL = "http://example.org/"
# [en, zh-cn, fr, ...] 设置默认的语言
defaultContentLanguage = "zh-cn"
# 网站语言, 仅在这里 CN 大写
languageCode = "zh-CN"
# 是否包括中日韩文字
hasCJKLanguage = true
# 网站标题
title = "我的全新 Hugo 网站"

# 更改使用 Hugo 构建网站时使用的默认主题
theme = "LoveIt"

[params]
  # LoveIt 主题版本
  version = "0.2.X"

[menu]
  [[menu.main]]
    identifier = "posts"
    # 你可以在名称 (允许 HTML 格式) 之前添加其他信息, 例如图标
    pre = ""
    # 你可以在名称 (允许 HTML 格式) 之后添加其他信息, 例如图标
    post = ""
    name = "文章"
    url = "/posts/"
    # 当你将鼠标悬停在此菜单链接上时, 将显示的标题
    title = ""
    weight = 1
  [[menu.main]]
    identifier = "tags"
    pre = ""
    post = ""
    name = "标签"
    url = "/tags/"
    title = ""
    weight = 2
  [[menu.main]]
    identifier = "categories"
    pre = ""
    post = ""
    name = "分类"
    url = "/categories/"
    title = ""
    weight = 3

# Hugo 解析文档的配置
[markup]
  # 语法高亮设置 (https://gohugo.io/content-management/syntax-highlighting)
  [markup.highlight]
    # false 是必要的设置 (https://github.com/dillonzq/LoveIt/issues/158)
    noClasses = false

注意

在构建网站时, 你可以使用 --theme 选项设置主题. 但是, 我建议你修改配置文件 (config.toml) 将本主题设置为默认主题.
